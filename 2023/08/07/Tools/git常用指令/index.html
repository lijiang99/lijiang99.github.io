Git常用指令

Author: li jiang Date: August 7, 2023  0:00:00 Category: 工具

1. git init

在项目的根目录下创建一个.git子目录，用于保存版本信息。

2. git add –all

将文件保存为二进制对象和更新暂存区。

3. git status

显示当前仓库状态，即输出哪些文件已修改，哪些文件已暂存等信息。

4. git commit -m “message”

将暂存区的文件提交至版本历史记录中，并附带一条描述信息。同时提交目录结构和说明，生成快照（保存当前目录结构，以及每个文件对应的二进制对象）。

5. git log

查看整个版本历史，即当前分支的所有快照。所谓分支就是指向某个快照的指针，分支名即指针名。git有一个特殊的指针HEAD，总是指向当前分支的最近一次快照。

6. git clone url

复制url所指向的远程仓库的所有代码和历史记录，并在本地创建一个与远程仓库相同的仓库副本。

7. git branch [dev]

创建名为dev的分支，若不带参数，则git branch可以查看当前仓库的所有分支。

8. git branch -d dev

删除名为dev的分支

9. git checkout dev

从当前分支切换到名为dev的分支，若携带-b参数（git checkout -b [dev]）则会先新建一个名为dev的分支并切换到该分支。相当于git branch dev + git checkout dev。

10. git merge dev

合并名为dev的分支到主分支上去，可能会出现合并冲突，需要手动去修改它。

11. git remote -v

列出当前仓库中已配置的远程仓库，并显示对应的url，不加-v，则只列出远程仓库名。

git remote show origin可显示指定的远程仓库（origin）的详细信息，包括url和跟踪分支。

12. git remote add origin url

向当前仓库添加一个名为origin的远程仓库，且对应的url为远程仓库的地址。

13. git fetch origin

从名为origin的远程仓库拉取代码库，获取远程有而本地没有的数据。
